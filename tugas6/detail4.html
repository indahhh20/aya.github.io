Project Overview

Proyek ini merupakan eksplorasi komprehensif tentang dasar-dasar jaringan komputer, dengan fokus pada IP address (Internet Protocol) dan infrastruktur jaringan yang mendukung koneksi berkecepatan tinggi seperti fiber optik. Berdasarkan pengalaman saya sebagai mahasiswa UIN Palu dan lulusan SMK 3 Sigi, proyek ini mencakup konfigurasi praktis, troubleshooting, dan integrasi dengan teknologi telekomunikasi modern. Tujuan utamanya adalah memahami bagaimana IP address mengatur komunikasi data di jaringan, serta membangun infrastruktur yang efisien untuk aplikasi kampus dan industri.

Konfigurasi IP Address

Langkah-langkah detail konfigurasi IP address untuk jaringan yang stabil:

  • Penentuan Alamat IP: Menggunakan IP statis (misalnya, 192.168.1.1 untuk router) atau dinamis via DHCP server untuk alokasi otomatis pada perangkat seperti komputer dan printer.
  • Subnetting: Membagi jaringan besar menjadi sub-jaringan kecil, contoh: Jaringan 192.168.0.0/24 dibagi menjadi 192.168.1.0/25 dan 192.168.1.128/25 untuk efisiensi bandwidth.
  • Gateway dan DNS: Konfigurasi default gateway (misalnya, 192.168.1.254) untuk routing keluar, dan DNS server (seperti 8.8.8.8 dari Google) untuk resolusi nama domain.
  • Pengujian Koneksi: Menggunakan tools seperti ping (ping 192.168.1.1) dan traceroute untuk memverifikasi konektivitas dan latency.

Infrastruktur Jaringan

Pembangunan infrastruktur jaringan yang terintegrasi dengan fiber optik:

  • Instalasi Switch dan Router: Menggunakan switch unmanaged/managed (misalnya, Cisco 2960) untuk distribusi data lokal, dan router (seperti MikroTik) untuk routing antar jaringan.
  • Integrasi Fiber Optik: Menghubungkan kabel fiber optik ke media converter atau SFP module pada switch untuk transmisi data berkecepatan tinggi (1Gbps atau lebih).
  • VLAN dan Segmentasi: Implementasi Virtual LAN (VLAN) untuk memisahkan traffic, contoh: VLAN 10 untuk admin, VLAN 20 untuk mahasiswa, menggunakan protokol 802.1Q.
  • Monitoring dan Keamanan: Menggunakan software seperti Wireshark untuk analisis paket, dan firewall (misalnya, pada router) untuk mencegah intrusi.

Hasil dan Aplikasi

Dampak praktis dari konfigurasi dan infrastruktur ini:

  • Peningkatan Performa: Kecepatan jaringan meningkat hingga 50% di area uji coba kampus, dengan latency rendah (<10ms) berkat integrasi fiber optik.
  • Reduksi Downtime: Troubleshooting efektif mengurangi waktu henti jaringan sebesar 30%, melalui identifikasi masalah seperti IP conflict atau kabel rusak.
  • Aplikasi Praktis: Diterapkan dalam proyek PKL untuk penyambungan fiber optik, serta simulasi jaringan kampus menggunakan software seperti Packet Tracer.
  • Pengetahuan Tambahan: Memperdalam pemahaman tentang protokol TCP/IP, routing statis/dinamis, dan keamanan jaringan, yang mendukung karir di IT dan telekomunikasi.
